5.1Was ist ein Algorithmus?

Algorithmen begegnen uns ständig: beim Onlinebanking, im Routenplaner, beim Online-Shopping, bei der Internetsuche. Aber auch außerhalb der Informatik – Bedienungsanleitungen, Spielregeln oder Anwendungshinweise für Arzneimittel sind nichts anderes als Algorithmen in normaler Sprache.

📌 Merke: Algorithmus

Ein Algorithmus ist eine Folge von Handlungsanweisungen, die eindeutig den Weg zur Lösung eines gegebenen Problems beschreiben.

Eigenschaften von Algorithmen

Damit eine Handlungsvorschrift wirklich ein Algorithmus ist, muss sie diese Eigenschaften erfüllen:

EigenschaftBedeutungGegenbeispiel (verletzt die Eigenschaft)
Allgemeingültigkeit Der Algorithmus löst eine ganze Klasse gleichartiger Probleme, nicht nur einen Einzelfall. Ein Primzahl-Algorithmus, der nur für den Bereich 3–100 funktioniert.
Eindeutigkeit Die Anweisungen dürfen sich nicht widersprechen. „Zwiebel in Würfel schneiden“ – aber später: „Zwiebelringe in die Pfanne geben“.
Ausführbarkeit Jede einzelne Anweisung muss für den Ausführenden (Mensch oder Computer) verständlich und ausführbar sein. „Wasser auf 120 °C erhitzen“ – geht (im offenen Topf) nicht.
Finitheit Die Beschreibung besteht aus endlich vielen Anweisungen mit begrenzter Zeichenzahl. „Addiere 1. Addiere 1. Addiere 1. …“ (unendlich aufgeschrieben)
Terminiertheit Der Algorithmus kommt nach endlich vielen Schritten zu einem Ende und liefert ein Ergebnis. zahl = 2, dann: „Wiederhole solange zahl > 0: zahl = zahl · 2“ – endet nie.
Determiniertheit Gleiche Eingaben liefern immer dasselbe Ergebnis. Die Anweisung „Bearbeite die Hausaufgaben“ – das Ergebnis fällt jedes Mal anders aus.
Determinismus Zu jedem Zeitpunkt gibt es höchstens eine Möglichkeit fortzufahren – der Folgeschritt ist eindeutig. Ein Rezept mit mehreren Würz-Varianten ohne Vorgabe, welche zu wählen ist.

5.2Warum ist Sortieren so wichtig?

  • Bis zu ¼ der Rechenzeit von Großrechnern entfällt auf Sortierung (Studie der Stanford University).
  • Mehr als 40 verschiedene Sortierverfahren sind bekannt und werden genutzt.
  • Sortierte Daten lassen sich viel schneller durchsuchen (z. B. Stichwortverzeichnis) – das sehen Sie in Abschnitt 5.6 selbst.
  • Sortieren ist Bestandteil zahlreicher weiterer Algorithmen und dient der Visualisierung von Daten (z. B. Medaillenspiegel bei Olympia).
ℹ️ Best Case und Worst Case

Um Sortieralgorithmen zu bewerten, vergleicht man oft den Best Case (bester/schnellster Fall) mit dem Worst Case (schlechtester/langsamster Fall): Wie lange dauert die Sortierung mindestens – und wie lange höchstens?

5.3Bubble Sort

Der Bubble Sort ist ein vergleichsbasierter Sortieralgorithmus. Sein Prinzip: Benachbarte Elemente werden verglichen und bei Bedarf getauscht – die großen Werte steigen dabei wie Luftblasen (englisch bubbles) ans Ende des Arrays auf.

📌 Das Prinzip in vier Schritten
  1. Starte mit dem ersten Array-Element. Ist es größer als sein rechter Nachbar, werden beide getauscht.
  2. Fahre mit dem zweiten Element fort – wieder: Ist es größer als der rechte Nachbar, tauschen.
  3. Wiederhole das bis zum Ende des Arrays. (Ein kompletter Durchlauf heißt Bubble-Phase.)
  4. Wiederhole die Bubble-Phasen, bis das Array sortiert ist. Das jeweils letzte Element der vorherigen Phase muss nicht mehr betrachtet werden – das größte Element ist ja bereits ans Ende „aufgestiegen“.

Bubble-Phase 1 von Hand (Array [5, 4, 9, 1, 3])

Vergleicha[0]a[1]a[2]a[3]a[4]
Start54913
5 > 4? Ja → Tausch45913
5 > 9? Nein → kein Tausch45913
9 > 1? Ja → Tausch45193
9 > 3? Ja → Tausch45139

Ende der ersten Bubble-Phase: Die 9 ist ganz nach hinten „aufgestiegen“ – ab a[4] ist das Array sortiert. Die weiteren Phasen laufen genauso, nur jeweils ein Feld kürzer: Nach Phase 2 ist [4, 1, 3, 5, 9], nach Phase 3 [1, 3, 4, 5, 9] – Phase 4 stellt fest, dass nichts mehr getauscht wird: fertig.

Implementierung in Python

zahlen = [5, 4, 9, 1, 3]
laenge = len(zahlen)
anzahl_bubble_phasen = 0
tausch = True

while tausch:
    tausch = False
    anzahl_bubble_phasen = anzahl_bubble_phasen + 1
    for i in range(0, laenge - anzahl_bubble_phasen):
        if zahlen[i] > zahlen[i + 1]:
            zwischenspeicher = zahlen[i]
            zahlen[i] = zahlen[i + 1]
            zahlen[i + 1] = zwischenspeicher
            tausch = True

print(zahlen)
print("Anzahl der Bubble-Phasen:", anzahl_bubble_phasen)
[1, 3, 4, 5, 9]
Anzahl der Bubble-Phasen: 4

So arbeiten die Bausteine zusammen:

  • Die äußere while-Schleife startet so lange neue Bubble-Phasen, wie in der letzten Phase noch getauscht wurde (Merkvariable tausch).
  • Die innere for-Schleife ist eine Bubble-Phase: Sie vergleicht Nachbarn und tauscht mit dem bekannten Zwischenspeicher-Muster aus Kapitel 4.
  • laenge - anzahl_bubble_phasen verkürzt jede Phase um ein Feld – der bereits sortierte hintere Teil wird nicht mehr angefasst.

5.4Selection Sort

Der Selection Sort (englisch selection = Auswahl) geht intuitiver vor – so würden viele Menschen Spielkarten sortieren:

📌 Das Prinzip
  1. Suche das kleinste Element und tausche es mit dem ersten Element.
  2. Suche das zweitkleinste Element (im Rest-Array) und tausche es mit dem zweiten Element.
  3. Und so weiter – bis zum vorletzten Element. (Das letzte ist dann automatisch richtig.)

Ablauf von Hand (Array [5, 4, 9, 1, 3])

SchrittArray danach
Kleinstes (1) mit a[0] tauschen[1, 4, 9, 5, 3] – bis Index 0 sortiert
Zweitkleinstes (3) mit a[1] tauschen[1, 3, 9, 5, 4] – bis Index 1 sortiert
Nächstes (4) mit a[2] tauschen[1, 3, 4, 5, 9] – bis Index 2 sortiert
Nächstes (5) ist schon an a[3][1, 3, 4, 5, 9] – bis Index 3 sortiert → fertig!

Implementierung in Python

zahlen = [5, 4, 9, 1, 3]
laenge = len(zahlen)

for akt_idx in range(laenge - 1):
    min_idx = akt_idx
    for j in range(akt_idx + 1, laenge):
        # Suche nach der niedrigsten Zahl
        if zahlen[j] < zahlen[min_idx]:
            min_idx = j
    # Tausch
    if akt_idx != min_idx:
        zwischenspeicher = zahlen[akt_idx]
        zahlen[akt_idx] = zahlen[min_idx]
        zahlen[min_idx] = zwischenspeicher

print(zahlen)
[1, 3, 4, 5, 9]
  • Die äußere Schleife läuft über die Zielposition akt_idx (0, 1, 2, …).
  • Die innere Schleife sucht im Rest-Array den Index des kleinsten Elements (min_idx) – das ist das Such-Muster aus Kapitel 4.6, nur dass hier der Index gemerkt wird statt des Werts.
  • Danach wird höchstens einmal pro Durchlauf getauscht – anders als beim Bubble Sort, der in jeder Phase viele Nachbar-Täusche machen kann.
ℹ️ Array komplett ausgeben

Für die Kontrolle können Sie ein Array elementweise mit der for-Schleife ausgeben – oder als Ganzes mit print(zahlen). Beides haben Sie in Kapitel 4.6 gelernt.

Implementierung: Einlass-Kontrolle im Verein

# Mitgliedsnummern
mnr = [1001, 1019, 1014, 1009, 1005, 1002, 1018, 1008, 1003, 1010,
       1007, 1004, 1020, 1013, 1015, 1011, 1017, 1012, 1006, 1016]

# Eingabe
nummer = int(input("Mitgliedsnummer eingeben: "))

laenge = len(mnr)
gefunden = False

# Suche
for i in range(laenge):
    if mnr[i] == nummer:
        gefunden = True

# Ausgabe
if gefunden:
    print("Zutritt gewährt")
else:
    print("Zutritt verweigert")
Mitgliedsnummer eingeben: 1013
Zutritt gewährt
📌 Merke

Die lineare Suche funktioniert bei jedem Array – sortiert oder unsortiert. Ihr Preis: Im Worst Case (Wert ganz hinten oder gar nicht vorhanden) muss jedes einzelne Element geprüft werden.

5.6Die binäre Suche

Die binäre Suche ist deutlich effizienter – sie funktioniert aber nur auf einem sortierten Array. Ihr Trick: Der Suchbereich wird mit jedem Vergleich halbiert (so wie Sie in einem Wörterbuch auch nicht vorn anfangen, sondern in der Mitte aufschlagen).

📌 Vorgehensweise
  1. Setze den Start s auf das erste und das Ende e auf das letzte Element im Array.
  2. Ermittle die Mitte m zwischen Start und Ende (ggf. abrunden).
  3. Ist das gesuchte Element kleiner als Element m → setze e auf m − 1.
  4. Ist es größer → setze s auf m + 1.
  5. Ist es gleich → Suche erfolgreich!
  6. Wiederhole ab Schritt 2 (solange s ≤ e).

ℹ️ Abrunden mit int()

Die Mitte (s + e) / 2 ist oft keine ganze Zahl. In Python wird eine Dezimalzahl abgerundet, indem man sie in eine Ganzzahl umwandelt: int(7.98) → 7, int(9.07) → 9.

Implementierung: Losnummern-Prüfung

ln_gewinne = [11586, 12569, 22254, 23654, 32541,
              55589, 85665, 87512, 98521, 98741]

losnr = int(input("Gesuchte Losnummer: "))
gefunden = False
index_anfang = 0
index_ende = len(ln_gewinne) - 1

while gefunden == False and index_anfang <= index_ende:
    index_mitte = int((index_anfang + index_ende) / 2)

    if ln_gewinne[index_mitte] == losnr:
        gefunden = True
    elif ln_gewinne[index_mitte] > losnr:
        index_ende = index_mitte - 1
    else:
        index_anfang = index_mitte + 1

if gefunden:
    print("Sie haben gewonnen :-)")
else:
    print("Sie haben leider nicht gewonnen :-(")
Gesuchte Losnummer: 85665
Sie haben gewonnen :-)

Die while-Schleife läuft, solange nichts gefunden wurde und der Suchbereich nicht leer ist (index_anfang <= index_ende). Ist der Suchbereich leer, kommt die gesuchte Nummer nicht vor.

5.7Algorithmen im Vergleich

AlgorithmusIdee in einem SatzWann geeignet?
Bubble Sort Nachbarn vergleichen und tauschen – Große steigen wie Blasen nach hinten auf. Einfach zu verstehen und zu implementieren; erkennt am Tausch-Merker, wenn früh fertig sortiert ist.
Selection Sort Immer das kleinste Element des Rests auswählen und nach vorn tauschen. Intuitive Vorgehensweise; pro Durchlauf höchstens ein Tausch.
Lineare Suche Jedes Element der Reihe nach prüfen. Einziger Weg bei unsortierten Arrays; bei kleinen Arrays völlig ausreichend.
Binäre Suche Suchbereich per Mitte-Vergleich immer wieder halbieren. Sehr schnell bei großen, sortierten Arrays (1 000 000 Elemente → max. ca. 20 Vergleiche!).

Übung 3 · Selection Sort absteigend schwer

Für eine Bestenliste sollen fünf eingegebene Punktzahlen absteigend (größte zuerst) sortiert werden. Implementieren Sie den Selection Sort – überlegen Sie zuerst, welche einzige Stelle des Algorithmus sich gegenüber der aufsteigenden Variante ändert.

Original: L2_2.2.3 Arbeitsauftrag Selection Sort absteigend (BPE 7)

Lösung anzeigen 
punkte = []
for i in range(5):
    punkte.append(int(input("Punktzahl: ")))

laenge = len(punkte)

for akt_idx in range(laenge - 1):
    max_idx = akt_idx
    for j in range(akt_idx + 1, laenge):
        # Suche nach der GRÖSSTEN Zahl
        if punkte[j] > punkte[max_idx]:
            max_idx = j
    if akt_idx != max_idx:
        zwischenspeicher = punkte[akt_idx]
        punkte[akt_idx] = punkte[max_idx]
        punkte[max_idx] = zwischenspeicher

print(punkte)

Nur der Vergleichsoperator in der inneren Schleife dreht sich um: Statt des kleinsten wird das größte Element gesucht (> statt <) und nach vorn getauscht.

Übung 6 · Welcher Algorithmus passt? klausurtypisch

Begründen Sie jeweils kurz, welcher Suchalgorithmus geeignet ist:

  1. In einem unsortierten Array mit 30 Teilnehmernamen soll ein Name gefunden werden.
  2. In einem aufsteigend sortierten Array mit 1 000 000 Kundennummern soll eine Nummer gefunden werden.
  3. In einem sortierten Array mit 10 Losnummern soll jede Nummer daraufhin geprüft werden, ob sie doppelt vorkommt.

Original: L2_3.3.2 Arbeitsauftrag Suchalgorithmen prüfen (BPE 7)

Lösung anzeigen
  1. Lineare Suche – das Array ist unsortiert, die binäre Suche ist daher nicht anwendbar. Bei nur 30 Elementen ist sie zudem völlig ausreichend schnell.
  2. Binäre Suche – das Array ist sortiert und sehr groß: Statt bis zu 1 000 000 Vergleichen (linear) braucht die binäre Suche höchstens ca. 20.
  3. Hier hilft keine einzelne Suche: Sinnvoll ist ein Durchlauf mit Vergleich benachbarter Elemente – in einem sortierten Array stehen Duplikate direkt nebeneinander. (Erkenntnis: Sortierung macht auch andere Aufgaben einfacher!)
